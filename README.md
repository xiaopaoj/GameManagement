# 游戏管理软件

面向 Windows 的单机游戏原始压缩文件、游戏盘和存档统一管理工具。

当前版本：`1.0.0`

> 版本 1.0.0 已完成全部计划阶段与最终测试验收。

## 项目目标

- 管理分散在多个目录中的 ZIP、RAR 游戏原始文件。
- 将当前游玩的游戏准备到用户选择的固态游戏盘。
- 支持两级解压、独立密码保存、游戏目录和主游戏 EXE 识别。
- 对游戏目录和用户确认的 Windows 系统存档目录进行存档管理。
- 支持普通归档、特殊归档、版本切换和每日计划备份。
- 最终发布为无需额外安装运行时的单文件 Windows EXE。

完整需求参见：[游戏管理软件需求与实施计划_v1.0.0.md](./游戏管理软件需求与实施计划_v1.0.0.md)。

## 当前已实现

- `.NET 8 + WPF` 中文桌面应用工程。
- EXE 所在目录的 `data`、`logs` 自动创建与写入权限检查。
- 主数据库无论是否开启安全密码模式都以 AES-256-GCM 密文持久化并原子替换写入；随机主密钥在普通模式下由当前 Windows 用户 DPAPI 保护。旧版明文数据库迁移并回读校验后，只有经用户确认才永久删除。
- 设置页可开启、修改或关闭安全密码模式；开启后使用 PBKDF2-SHA256 派生密钥包装主数据密钥，启动时必须先解锁，未解锁的计划备份禁止执行。关闭安全模式后数据库仍保持密文。
- 安全模式未解锁时仅显示不加载真实数据的最小包装外壳；支持立即锁定、可配置空闲自动锁定和持久化密码错误等待。运行中的软件任务会阻止锁定，避免中断复制、解压或归档。
- 游戏图标和运行日志均使用主数据密钥加密存储；图标只在解锁后的内存中显示并在锁定时清除缓存，日志通过软件内置查看器解密查看。旧明文图标和日志完成迁移及回读校验后，仍需用户确认才会永久删除。
- 多扫描路径管理。
- 扫描路径直属 ZIP、RAR 文件和直属游戏文件夹；文件夹内部压缩包延迟到“准备游玩”阶段递归识别，避免机械硬盘扫描时读取大量文件头尾。
- 文件扫描显示候选统计进度、百分比和当前路径，扫描结束前使用阻塞遮罩禁用其他界面操作并阻止关闭主窗口。
- 扫描候选中的路径有效状态和已添加状态使用只读单向绑定，避免扫描结果渲染时触发 WPF 双向写入异常。
- 隐藏已添加候选项目。
- 候选项目多选批量添加，自动生成游戏名称和“初始版本”；来源指纹计算在后台运行且不锁定主窗口。
- 候选项目支持按名称或路径、来源类型、磁盘和添加状态筛选。
- 添加游戏和新增版本时记录来源文件数量、总体积、修改时间和 SHA-256 指纹，目录来源生成聚合指纹。
- 游戏详情支持在同一个窗口中修改显示名称和备注，并显示原始路径有效状态。
- 游戏详情的信息区和版本列表支持滚动及拖动分隔线调整高度，操作按钮过多时在独立区域滚动；通用输入窗口会按内容自适应高度，避免高 DPI 或主题内边距导致控件挤压变形。
- 版本管理支持添加用户命名版本、编辑版本名称与备注、查看来源、切换当前版本及根据文件名、大小、修改时间和 SHA-256 结果重新定位来源。
- 多游戏盘配置及标准工作目录创建。
- 游戏盘支持默认项、启用状态和最低保留空间设置，并显示路径状态、磁盘类型、总容量及剩余空间。
- 游戏库、任务中心、打开目录和启动入口。
- 游戏库右键菜单直接进入具体功能，不再先显示详情页；游戏库支持多选，多选时可批量应用解压模板，并将准备或归档加入串行后台队列；后台队列使用不创建 XAML/Owner/窗口句柄的无界面进度接收器，不显示或闪现公共进度窗口，只在确需人工输入时显示对应窗口；批次内相同的盘符、解压流程、密码、风险确认和归档清理选项只询问一次并应用到全部所选游戏，复用项无效时才单独重选。
- 游戏库右键操作导致游戏信息变化时自动刷新列表，准备完成或删除记录后无需手动点击刷新。
- 支持集中维护两步解压密码模板并按游戏绑定，模板密码由 Windows DPAPI 加密保存；游戏库右键通过动态二级菜单直接切换模板。
- 设置页支持即时切换并持久化“经典主题”和“Windows 11 风格”，后者统一适配主窗口标签页、按钮、表格边框与行状态，以及带阴影的圆角菜单和各功能弹窗。
- 设置页可选择“自动选择 / WinRAR / 内置解压”：自动模式检测用户已安装的 WinRAR 并优先静默后台调用，未安装时回退内置引擎；WinRAR 模式会自动传入已保存密码并支持任务取消。WinRAR 需由用户自行安装，密码在子进程运行期间会作为瞬时命令行参数传递。
- 设置页提供可持久化维护的 EXE 忽略清单；支持新增、编辑、删除具体文件名，游戏目录和启动文件扫描会忽略命中项。
- 游戏库先显示未归档游戏、最后显示已归档游戏，各组内部按添加时间和上次游玩时间倒序排列；文件扫描候选默认按最后修改时间倒序排列。
- 持久化任务中心记录扫描、来源指纹和准备游玩任务的状态、进度、当前路径、错误及临时目录。
- 所有可后台运行任务的公共进度窗口均支持系统最小化和“后台运行”，最小化后任务继续执行并可从任务栏或任务中心查看。
- 扫描、来源指纹和准备游玩支持取消；软件异常退出后将未完成任务恢复为“已中断”。
- 失败、取消和中断任务保留临时目录，支持查看及一次确认安全清理；成功准备任务自动清理临时目录。
- 任务中心可对失败、取消或中断的准备任务选择“重新发起准备”或“从失败处继续准备”；继续操作只复用能够验证出有效游戏目录的结果并重新建立基线，运行任务期间禁止刷新数据。
- 准备任务直接在 `Games/{游戏ID}/.prepare` 内复制和解压，建立基线后通过同盘目录移动提交到 `Games/{游戏ID}/content`，避免大型游戏再次全量复制；失败时不会写入可游玩路径。
- 批量准备或归档时，主窗口顶部按整个队列显示总进度；准备阶段发现多个启动文件时延迟到首次启动再由用户确认。
- 支持通过“查看详情”按钮进入游戏详情，双击游戏库条目直接启动游戏；游戏行右键菜单复用详情页的操作与确认逻辑，并可直接勾选游戏是否存在 Windows 系统目录存档。
- 已为需求定义的准备游玩、归档、特殊归档、备份、存档、版本、重新定位和删除原始文件建立可发现且可执行的入口。
- 主导航已包含游戏库、文件扫描、设置、任务中心、存档管理、备份管理和操作历史。
- 自包含单文件 EXE 发布配置。
- 自动化测试覆盖扫描边界、取消、任务序列化与中断恢复、临时目录安全边界、来源指纹变更、压缩内容空间估算、存档、备份、归档和删除规则。
- “准备游玩”完整流程：压缩条目内容感知空间估算与安全回退、游戏盘选择、复制来源、两级压缩文件选择、ZIP/RAR 头尾区域识别、按引擎处理混合文件（内置引擎剥离伪装前缀并生成规范化临时文件，WinRAR 直接读取原始混淆文件）、最大文件 ZIP/RAR 依次尝试、DPAPI 密码保存、递归确定首个有效 EXE 所属游戏目录、展示目录直属 EXE/index.html 和 SHA-256 文件基线。
- 所有阻塞式进度任务和临时禁用窗口的异步任务结束前，统一隐藏进度界面、恢复调用窗口并使用由调用窗口持有的结果提示；提示关闭后再次恢复窗口前台状态，覆盖准备游玩、快照、备份、密码验证、特殊归档、来源校验和启动前系统存档扫描等入口。
- 支持 `.partNN.rar`、`.rar/.r00`、`.zip.001`、`.z01/.zip` 和 `.7z.001` 分卷组聚合、整组复制和可推断缺失卷检查；7z 会读取首卷头部声明的完整归档最低长度，可在只有 `.001` 时发现缺少末尾 `.002` 等分卷；缺失时经人工确认只尝试一次。
- 解压密码管理支持按版本和步骤查看、修改、一次确认删除及重新验证，密码继续使用 Windows DPAPI 加密；解压密码窗口存在历史记录时默认展开竖向历史列表。
- 同一版本再次准备时可重放第一次、第二次压缩文件选择、保存密码和最大文件 ZIP/RAR 降级格式；第一次确认可勾选自动重放本次任务的后续历史步骤，未勾选时第二次仍单独确认。
- 游戏目录按稳定顺序递归识别，启动候选根据目录同名、体积、常见关键词、版本信息及辅助程序排除名单评分，仍由用户最终确认。
- 自动识别失败或推荐不正确时，可在解压结果或可游玩目录范围内手动选择 EXE/index.html；游戏详情中的“识别游戏目录”已经支持独立重新识别。
- 主 EXE 图标保存到 `data/cache/icons` 并显示在游戏库和游戏详情中。
- Windows 系统目录存档扫描默认关闭，用户可在游戏详情或游戏库右键菜单中按游戏明确启用。
- 已确认并保存到 `GameSave/current` 的游戏目录存档，后续退出扫描通过大小和修改时间快速判断是否变化；未变化文件不再重复计算 Hash 或生成归档候选。存档候选树的关闭按钮同时支持自动弹出的非模态窗口。
- 启动游戏后以所选启动进程记录运行状态，禁止同一游戏重复启动；退出后记录退出时间、退出代码和持续时长，软件重启后可恢复仍有效的进程监听。
- 首次准备保存包含游戏、版本、相对路径、大小、时间、SHA-256 和文件类别的文件基线；后续先按路径、大小和时间快速比较，必要时计算 Hash。
- 游戏目录新增、修改和删除内容会生成存档候选；首次基线文件、≥100 MB 文件和疑似资源包默认排除，但允许人工确认，具体文件排除规则会持久化。
- 主程序退出后自动扫描变化；已确认的新增或修改存档自动更新 `GameSave/{游戏ID}/current` 并创建正常或异常快照，首次候选、默认排除项和删除变化弹窗确认。
- current 与快照使用临时目录、文件清单和 SHA-256 校验后提交，失败时回滚；内容未变化时不生成重复快照。
- 正常和异常快照分别保留三个，超额只标记建议清理；支持查看清单、文本/图片预览、定位、一次确认删除及人工恢复说明。
- 首次运行前扫描 AppData、LocalAppData、LocalLow、Documents 和 Saved Games，并剪枝排除明显属于其他软件及隐藏内容的目录；主 EXE 退出后只定位运行期间新建且与游戏名称或主 EXE 匹配的存档文件夹。成功后保存目录等待归档时处理，失败后不再自动扫描并在归档时询问用户。
- 存档候选使用支持多选的文件树展示，勾选文件夹会同时选择其全部内容。
- 系统存档目录支持添加、修改、启用、禁用、删除配置和共享关联；共享目录变化始终要求人工确认游戏归属。
- 系统目录候选与游戏目录候选统一写入 current 和正常/异常快照，清单保留原始绝对路径，启动前扫描支持进度、取消和缓存自动清理。
- 普通归档会复扫游戏目录存档并校验 current、系统扫描状态和匹配内容指纹的有效外部 ZIP；归档状态与游戏目录回收站清理状态独立保存。
- 再次准备支持跨游戏盘复制 current、保留来源、覆盖目标、Hash 校验，并按清单事务式恢复游戏目录和 Windows 系统目录存档。
- 版本管理支持“切换并准备”，按当前版本归档、目录回收站清理、目标版本切换和重新解压顺序执行，失败时标记“版本切换失败”且不自动恢复旧版本。
- 单游戏手动备份生成并校验无密码 ZIP，包含 current、清单、游戏版本、来源盘、原始恢复路径和文件 Hash；有效备份会解除普通归档的外部备份前置阻塞。
- 每日计划备份通过同一单文件 EXE 的无界面模式合并所有游戏盘完整 `GameSave`，内容未变化不重复生成；备份盘离线时等待上线后补执行并发送 Windows 通知。
- 外部备份列表支持只读内容、恢复路径和人工恢复说明、系统打开、重新校验、一次确认删除；完整计划备份只保留三个标记，超额不自动删除。
- 特殊归档可在 GameSaveTemp 从原始 ZIP/RAR 构建干净基线并比较混乱目录；无法建立基线时只允许人工多选文件，完成本地快照和外部备份前禁止删除目录。
- 原始压缩包及分卷通过独立入口二次确认并只进入 Windows 回收站；删除游戏主记录时可按复选框同步清理版本、可游玩目录、本地存档、系统配置、外部备份和临时目录，完整游戏名称以可复制文本展示并仍需粘贴或输入确认，原始压缩包不会被此入口删除。
- 操作历史中的“删除历史”可查看回收站、永久删除和数据库记录删除结果。

## 已知限制

- 首层 ZIP、RAR 可读取条目元数据时，软件会按来源复制量、首层展开量、二次解压与最终目录预留及安全余量进行分项估算。
- 加密或混淆的二次压缩内容在实际完成首层解压前无法获得绝对精确的展开体积；元数据读取失败时按来源总体积四倍进行安全回退，最终仍以目标游戏盘实际剩余空间为准。

## 项目结构

```text
GameManagement/
├─ src/GameManagement.App/       # WPF 应用源码
├─ tests/GameManagement.Tests/   # 自动化测试
├─ tools/                        # 本地开发辅助脚本
├─ GameManagement.sln
├─ global.json
└─ 游戏管理软件需求与实施计划_v1.0.0.md
```

`build`、`bin`、`obj`、本地 SDK、运行数据库和日志不会提交到 Git。

## 开发环境

- Windows 10 或 Windows 11
- .NET 8 SDK
- Visual Studio 2022、JetBrains Rider 或 Visual Studio Code（可选）

如果系统未安装 .NET 8 SDK，可以将 SDK 安装到项目本地目录：

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\dotnet-install.ps1 -Version 8.0.100 -InstallDir .\tools\dotnet -NoPath
```

## 构建与测试

使用项目本地 SDK：

```powershell
.\tools\dotnet\dotnet.exe restore GameManagement.sln
.\tools\dotnet\dotnet.exe build GameManagement.sln -c Debug
.\tools\dotnet\dotnet.exe test GameManagement.sln -c Debug
```

如果已经全局安装符合 `global.json` 要求的 SDK，可以将命令中的 `.\tools\dotnet\dotnet.exe` 替换为 `dotnet`。

## 发布单文件 EXE

```powershell
.\tools\dotnet\dotnet.exe publish .\src\GameManagement.App\GameManagement.App.csproj `
  -c Release `
  -r win-x64 `
  --self-contained true `
  -o .\build\release
```

发布产物：

```text
build\release\GameManagement.exe
```

发布时若正式 EXE 未被占用，直接更新 `GameManagement.exe`；只有正式 EXE 正在运行或无法替换时才生成 `GameManagement.new.exe`，不强制结束用户进程。

## 运行数据

程序运行后在 EXE 所在目录创建：

```text
data/
logs/
```

这些目录包含用户配置、游戏记录、任务状态和运行日志，不应提交到 Git。
